# Descrição do caso escolhido
O projeto consiste em um site institucional para um Personal Trainer fictício chamado João Silva. O objetivo é simular a presença digital de um profissional da área de educação física, oferecendo informações sobre serviços, benefícios e formas de contato.

---

#  Necessidade identificada
Foi identificado que muitos profissionais autônomos da área de personal trainer não possuem um site estruturado para divulgação de seus serviços. Isso dificulta o acesso de novos clientes e reduz a visibilidade do profissional.

O site foi desenvolvido para resolver esse problema, criando uma página simples, objetiva e funcional para apresentação dos serviços e contato direto.

---

# Público-alvo
O público-alvo são pessoas interessadas em:
- emagrecimento
- hipertrofia (ganho de massa muscular)
- melhora do condicionamento físico

Esses usuários buscam informações rápidas e formas simples de contato com o profissional.

---

# Objetivo do site
O principal objetivo do site é facilitar o contato entre o usuário e o personal trainer, incentivando o agendamento de avaliações por meio do botão de WhatsApp.

---

#  Processo de desenvolvimento
O projeto foi desenvolvido em etapas:
1. Definição do tema (personal trainer)
2. Estruturação do HTML com seções semânticas
3. Estilização com CSS externo
4. Aplicação de responsividade com media queries
5. Implementação de JavaScript simples para interação (FAQ e botão de agendamento)
6. Uso de Bootstrap para organização de cards e layout responsivo
7. Testes de funcionamento em diferentes tamanhos de tela

---

# Principais decisões do projeto
- Uso de HTML semântico (header, section, footer)
- Separação de arquivos (HTML, CSS e JS)
- Uso de cores escuras com verde para identidade fitness
- Estrutura responsiva para dispositivos móveis
- Implementação de botão de WhatsApp como chamada principal para ação
- Uso de JavaScript simples para interação na FAQ

---

#  Funcionalidade em JavaScript
O projeto possui duas funcionalidades principais em JavaScript:

1. Exibição de mensagem ao clicar no botão "Agendar Avaliação"
2. Mostrar e ocultar as respostas da seção FAQ

Essas funcionalidades foram escolhidas por serem simples, mas melhorarem a interação do usuário com a página.

---

#  Uso de Bootstrap
O Bootstrap foi utilizado para facilitar a criação do layout responsivo, principalmente na seção de serviços, utilizando:
- container
- row
- col-md-4
- cards

Isso permitiu uma organização mais rápida e responsiva dos elementos.

---

# Testes realizados
O site foi testado em diferentes tamanhos de tela, verificando:
- responsividade do layout
- funcionamento do menu
- exibição correta das seções
- funcionamento do JavaScript
- carregamento de imagens e estilos
